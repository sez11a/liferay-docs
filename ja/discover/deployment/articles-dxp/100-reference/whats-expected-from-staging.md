# ステージングに期待される動作

[TOC levels=1-4]

ステージングプロセス中に、エンティティに何が起こるか疑問に思ったことはありますか。
ステージングは変更を認識しますか。その場合は、アップデートはどのように処理されますか。

この記事は、各Liferayエンティティのユースケースの一覧です。ユースケースの表は3つの列があります。

- *関連エンティティ*表のメインエンティティに関連または添付するエンティティたとえば、Webコンテンツの記事（メインエンティティ）テーブルでは、添付されたエンティティが構造になります。
- 実行されたアクション：関連エンティティで完了した変更のタイプ。
- ステージングは​​これをどのように処理しますか？：ステージングフレームワークが実行されたアクションを処理する方法。

目的のエンティティセクションに移動します。

## アプリケーション表示テンプレート（ADT）

次の表は、ADTによって特徴付けられたエンティティがステーイングフレームワークによってどのように処理されるかを記載しています。

この表に適用される関連エンティティは次の通りです。

- アセットパブリッシャー
- ブログ
- パンくず
- カテゴリーナビゲーション
- ドキュメントとメディア
- ランゲージセレクター
- ナビゲーション
- RSSパブリッシャー
- サイトマップ
- タグナビゲーション
- Wiki

| 関連エンティティ | 実行されたアクション | ステージングはこれをどのように処理しますか。 |
| :------------- | :--------------  | :----------------------------
| 上記リストのエンティティ | ADTがエンティティに追加されます | ADTが公開されます。ステージングによって、ADTと関連エンティティが個別に処理されます。 |
|  | エンティティのADTが調整されました | ADTが変更され、公開されました。 |
|  | エンティティのADTが削除されました | ADT削除が公開され、実稼働サイトから削除されます。 |
|  | 実稼働サイトの既存のADTがエンティティとして選択されます | エンティティポートレットとページが公開されました。エンティティが存在するページが公開されている場合、そのADTも公開されます。 |
|  | エンティティから参照されるADTが更新されます | ADTは変更済みとして公開され、公開されます。 |
|  | エンティティから参照されるADTは削除されます | 実稼働サイトでADTが削除され、エンティティはデフォルトのテンプレートにフォールバックします。 |
|  | ADTのエンティティがページから削除されました | ADTは影響を受けず、ページが公開されるときにエンティティと共に削除されます。 |

|  | ダイナミックセレクションがエクスポートされ、無効になります | コンテンツはAPで公開されませんが、以前公開されたコンテンツは影響されません。 |
|  | ダイナミックエクスポートの制限が変更されました | ダイナミックエクスポート制限で指定されたコンテンツが公開され、APに表示されます。 |
| マニュアルセット セレクション | APが公開されました | マニュアルセレクションに従ったコンテンツが公開されます。 |
|  | マニュアル セレクションが変更され、APが再度公開されます | マニュアルセレクションに従って公開されます。 |

|  | マニュアルセレクション エクスポートが無効です | APは何も公開しません。 |
|  | マニュアルエクスポートに制限がありません | APは、全く同じマニュアルで選択されたコンテンツを公開します。 |

## ブログ

次の表は、ブログに添付／関連するエンティティがステージングフレームワークによってどのように処理されるかを記載しています。

**注:** ブログエントリで使用できる画像が2種類あります。それらは、レガシーのイメージフレームワークとドキュメントライブラリフレームワークの2つの異なるフレームワークに由来します。ドキュメントライブラリ`FileEntry`は、新しいブログエントリ用に作成された画像エンティティです。レガシー画像フレームワークの`Image`エンティティは、画像を備えた古いブログエントリの下位互換性のために提供されています。
従来のブログではアップグレードプロセスが利用できないため、古いイメージエンティティと新しいイメージエンティティの両方がサポートされています。したがって、両方とも以下の表にリストされていますタイプを定義しない画像は、`FileEntry`タイプとして想定できます。

| 関連エンティティ | 実行されたアクション | ステージングはこれをどのように処理しますか。 |

| ブログエントリ | ブログエントリが作成されました | 新しいエントリが公開されます。 |
|  | ブログエントリがエクスポートされました | エクスポート元のサイトのエントリに変更はありません。 |
|  | ブログエントリはリファレンス無しでエクスポートされます。 | エクスポート元のサイトのエントリに変更はありません。 |

|  | ブログエントリがリファレンス無しでインポートされます | 変更されたエントリが公開されます。インポートする前にエントリで変更が行われた場合、結果のLARで`modifiedDate`フィールドにフラグが付けられます。つまり、エントリが新しいサイトにインポートされると、ステージングサイトでのみ使用でき、デフォルトでは実稼働サイトに公開されません。 |
